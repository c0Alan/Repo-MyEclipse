---------------------------------------------------------------
1.由于flash的xml机制，导致如果一个节点的子节点过多，
效率会严重降低。测试结果是：flash处理一百个子节点
需要耗时15ms左右，处理一千四百个子节点需要耗时2.7s。
所以建议高亮配置文件的每个关键字<c>节点控制在100个子节点左右

2.对应代码配置xml说明(待更新)
<def>
	<setup>
		<delimiters><![CDATA[.~!%^&*()-+=|/{}[]:;"'\<> ,?]]></delimiters>（分隔符）
		<sedelimiters><![CDATA[.~!%^&*()-+=|/{}[]:;"'\<> ,?]]></sedelimiters>（双击选中单词的分隔符）
		<commentseq>//</commentseq>（单行注释,最大两个）
		<blockcommenton>/*</blockcommenton>（多行注释开始，最大两个）
		<blockcommentoff>*/</blockcommentoff>（多行注释结束，最大两个）
		<de color="#808080"/>（分隔符的颜色）
		<comment color="#808080"/>（注释颜色）
		<string color="#009900" bold="false"/>（文本颜色）
		<default color="#009900"/>（预设的普通格式）
	</setup>
	<c color="#0000ff">（一组关键字）
		<w>#endinitclip</w>
	</c>
</def>

3.sql.xml里面设置的关键字必须为大写
